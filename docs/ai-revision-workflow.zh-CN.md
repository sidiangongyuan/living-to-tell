# AI 对比 / 修订工作流计划

中文 · [English](ai-revision-workflow.md)

这是一份面向产品的 AI 辅助编辑体验计划。目标是让 AI 建议更容易对比、更安全地应用，并让润色、扩写、续写、总结、诊断等任务有各自清晰的设计。

## 要解决的问题

1. 片段右侧信息栏里的 **润色** 会直接开始运行，但 AI 工作区里也有带参数的 **润色**，两者区别不清楚。
2. AI 工作区生成结果后，只能看到 AI 结果，不能同时看到原文。
3. 如果用户只选中片段中的一部分文字，界面需要明确说明：只发送选区，也只影响选区。
4. 应用 AI 后，用户不应该感觉自己的原文被覆盖掉。现在已有版本快照，但原文也应该在审阅流程里可见。
5. 润色、扩写、续写等任务现在共用太多 UI 状态，切换任务时容易继承不相关的参数。

## 产品原则

- **先预览，再写回**：任何会改动正文的 AI 结果，都必须先经过用户审阅。
- **原文与建议并排**：改写类任务应像 Markdown preview / diff 一样，同时显示原文和 AI 结果。
- **选区是明确目标**：有选区时，原文栏显示选区文本；长选区可折叠或打开大对比视图。
- **任务参数不串台**：从润色切到扩写或续写时，不应静默继承不相关的润色参数。
- **正文保持纯文本安全**：在引入真正的富文本 / 修订模式前，不要把脆弱的 HTML 或颜色标记写进正文。

## 目标体验

### 1. 片段右侧信息栏

把当前会立即运行的 **润色** 改成路由按钮：

- 标签：**AI 润色...** / **AI Polish...**
- 行为：
  - 打开 AI 工作区。
  - 绑定当前片段。
  - 如果正文里已有选区，默认目标为 **选区**。
  - 自动选择 **润色** 任务。
  - 展示可编辑参数，用户确认后再运行。
- 未来可以保留单独的 **快速润色**，但必须有可见默认预设和明确确认路径。

**收入作品** 仍然作为单独的整理 / 归档动作保留。

### 2. AI 结果对比区

改写类任务不再只显示一个结果框，而是显示并排对比：

| 左侧 | 右侧 |
| --- | --- |
| 原文，只读 | AI 结果，可在应用前编辑 |

规则：

- 目标是 **选区** 时，左侧只显示被选中的文字。
- 目标是 **整个片段** 或 **作品节块** 时，左侧显示完整源文本。
- **续写** 的左侧显示上下文结尾，右侧只显示新增续写内容。
- 诊断 / 问答类任务继续使用报告视图，不强行套用改写对比。
- 第一阶段可以先独立滚动；同步滚动以后再做。

### 3. 右侧信息栏里的选区预览

右侧信息栏宽度有限，不适合作为主 diff 区。它只显示紧凑状态：

- 当前目标：选区 / 整个片段 / 作品节块等。
- 选区字符数。
- 选区较短时显示摘要摘录。
- 选区较长时显示前几行，并提供 **打开 AI 工作区 / 打开对比**。

真正的原文-结果对比应放在 AI 工作区或对比弹窗里。

### 4. 应用 AI 时保留原文

建议分阶段实现。

#### P0：不改数据库结构

- 默认输出保持 **仅预览**。
- 写回前继续保存版本快照。
- 应用前始终显示原文 / AI 对比。
- 提供明确按钮：
  - **替换选区** / **替换片段** / **替换节块**
  - **保存 AI 结果为新片段**
  - **复制结果**
- 不再使用泛泛的 **应用**，而是显示具体动作，避免误覆盖。

#### P1：关联 AI 版本

- 把 AI 输出保存为原片段的关联变体 / AI 卡片。
- 记录任务、模型、时间、目标范围、原文摘录等元信息。
- 允许从版本历史或片段信息栏重新打开原文-结果对比。

#### P2：可视化修订 / 富审阅模式

- 增加一个渲染视图：原文删除部分用弱色 + 删除线，AI 新增部分用高亮色。
- 在明确存储和导出策略前，主编辑器仍保持纯文本安全。
- 不把 provider 返回的 HTML 当成正文的权威格式保存。

## 各任务的差异化设计

每个任务应有自己的默认参数和独立状态。

| 任务 | 主要输入 | 关键控件 | 默认输出 | 审阅方式 |
| --- | --- | --- | --- | --- |
| 润色 | 选区 / 片段 | 强度、风格提示、保留声音 / 原意 | 仅预览 | 原文-改写并排 |
| 扩写 | 选区 / 片段 | 目标长度 / 倍率、扩写方向、保留事实 | 预览或保存为新片段 | 原文-扩写并排 |
| 续写 | 片段结尾 / 选中结尾 | 续写长度、方向、约束 | 之后支持插入 / 追加 | 上下文 + 新续写 |
| 风格迁移 | 选区 / 片段 | 风格画像、强度、保留作者声音 | 仅预览 | 原文-改写并排 |
| 总结 | 片段 / 作品 / 作品集 | 摘要长度、段落或要点 | 报告 / 新片段 | 原文 + 摘要 |
| 提纲 | 片段 / 作品 | 层级深度、格式 | 报告 / 新片段 | 原文 + 提纲 |
| 标题 | 片段 / 作品 | 标题数量、语气 | 报告 / 复制 | 候选列表 |
| 结构诊断 | 片段 / 作品 | 后续可加严重程度筛选 | 仅报告 | 问题卡片 |
| 一致性检查 | 作品 / 作品集 | 范围和参考附件 | 仅报告 | 问题卡片 + 引用 |
| 资料库问答 | 附件 | 问题和引用 | 仅报告 | 答案 + 引用 |

切换任务时，应恢复该任务自己的最近设置，或重置为该任务默认值。润色里的风格 / 强度不应静默带到续写里，除非用户明确选择复用。

## 实施阶段

### Phase 1 — 先澄清，再对比

- 右侧信息栏 **AI 润色...** 只打开 AI 工作区，不立即发请求。
- AI 工作区改写类任务增加原文 / 结果并排区域。
- 在结果 UI 中保留目标类型和选区元信息。
- 把泛用 **应用** 改成具体破坏性动作标签。
- 在面板打开期间，为每个任务保存独立参数状态。

### Phase 2 — 变体与可重新打开的对比

- 增加关联 AI 变体模型，或复用 AI 卡片保存生成结果。
- 用户可以保存 AI 变体而不替换原文。
- 从版本历史 / 片段信息栏重新打开对比视图。

### Phase 3 — 可视化 diff / 修订模式

- 增加删除线 + 高亮色的渲染 diff。
- 只有在存储和导出策略清楚后，再考虑富文本编辑器。

## Phase 1 验收标准

- 从片段信息栏点击 **AI 润色...** 不会立即开始 AI 请求，而是打开带参数的 AI 工作区。
- 如果进入 AI 工作区前已有选区，目标默认是 **选区**，原文对比栏只包含该选区。
- 改写类任务生成后，原文和 AI 结果同时可见。
- AI 结果在写回前仍可编辑。
- 写回按钮明确说明动作：替换选区、替换片段、替换节块。
- 从润色切到扩写或续写时，不会静默复用不相关的润色设置。
- 原有安全机制仍然有效：写回前快照、测试通过、仓库不提交凭据或私有数据。
