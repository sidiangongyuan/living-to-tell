# AI 配置排查

活着为了讲述把每条 AI 接入保存为相互独立的配置档案。单模型功能使用唯一默认档案；AI 修改只发送本轮实际勾选的档案。

## 先按档案流程检查

1. 打开 **设置 → AI**。
2. 用三步向导新建档案，或使用 **扫描本机配置**。
3. 运行 **全部本地检查**。它只确认档案引用的 Key 或 CLI 登录是否存在，不联系模型。
4. 勾选真正要验证的档案，再发送最小真实测试。真实测试可能使用 token 并产生供应商费用。
5. 查看保存的健康状态：**未测试**、**已通过**、**失败**、**配置已变更需重测**。

本地检查通过，不代表地址、账号、模型或计费分组可用。只有真实测试成功，才能证明这一组 provider / model / transport 当前可用。

## 接入方式

| 方式 | 软件实际使用什么 | 重点检查什么 |
| --- | --- | --- |
| OpenAI API | API Key 与官方地址 | Key、模型权限、账号计费和模型名称。 |
| OpenAI 兼容中转站 | 档案独立 Key、Base URL、模型和协议 | 中转文档里的地址、模型名、账号分组，以及要求 Chat Completions 还是 Responses。 |
| Codex | 本机已有 Codex 登录 | Codex 已登录并能在本机完成最小请求。软件不提供应用内 ChatGPT OAuth。 |
| Gemini API | Gemini API Key 或兼容中转档案 | Key/项目权限、模型可用性、地址和中转协议要求。 |
| Gemini CLI | 本机已有 Gemini CLI 登录 | `gemini` 已安装、已登录，并且桌面应用进程能够访问。 |
| OpenCode | 本机 `opencode auth login` 会话 | OpenCode 已安装、已认证，所选模型仍可用。 |

## 常见诊断

| 界面诊断 | 代表什么 | 怎么处理 |
| --- | --- | --- |
| 未配置 API Key | 档案指向空的或不存在的凭据来源。 | 编辑档案并在本机保存 Key，或修正凭据来源。 |
| 未找到本机登录 | Codex、Gemini CLI 或 OpenCode 没有可用登录。 | 先用对应工具登录，再重新做本地检查和真实测试。 |
| 登录检查超时 | 本机 CLI 没有及时回应。 | 在终端确认 CLI 能运行，并检查它是否卡在交互提示。 |
| 401 | 地址拒绝了认证。 | 核对这个档案自己的 Key 与 Base URL，不能借用另一个档案的 Key。 |
| 403 | 认证已到达服务，但账号没有模型或分组权限。 | 检查模型权限、中转账号分组、地区、额度或计费授权。账号权限拒绝不能靠切换协议解决。 |
| 404 | Base URL、API 路径或模型名无法识别。 | 对照供应商最新文档，检查是否重复拼接 `/v1`。 |
| 429 | 账号被限流或额度耗尽。 | 等待或检查额度；软件不会自动重发请求。 |
| 超时 | 请求可能已经发出，但本地等待时间内没有收到结果。 | 重试前先看供应商后台；远端请求可能仍在生成或已经计费。 |
| HTML / 网关页面 | 代理返回了网页错误，而不是 AI API 响应。 | 检查 Base URL、代理状态、认证网关和要求的 API 路径。 |

公共界面会清洗错误，不显示原始 HTML、traceback、密钥内容或本机凭据路径。

## 协议与中转站

OpenAI 兼容服务通常使用 **Chat Completions** 或 **Responses**。三步向导会选择合理默认值，并把协议放在高级设置里。迁移旧配置时应保留已经验证过的协议，不能只看模型名猜测。

如果认证成功，但 provider 提示类似 `model-name-shared` 分组无权限，说明请求已经到达中转站。这是账号权限问题，不应改用另一个档案的密钥来伪装通过。

## 长任务与费用安全

- 离开 AI 修改页不会停止当前任务，返回页面后查询原任务状态。
- 重连不会创建新任务，也不会重新发送 provider 请求。
- OpenAI 兼容请求默认在 120 秒后结束本机等待，并禁用 SDK 隐藏的自动重试。高级用户可设置 `WRITER_OPENAI_TIMEOUT_SECONDS`；调整本机等待时间仍不能保证远端停止生成或计费。
- 本地中断只停止等待和采用后续结果，不能保证已发出的远端请求停止生成或停止计费。
- 一个模型失败不会阻塞其他已选模型的成功结果。
- 运行后文章发生变化时，位置写回会被阻止。可以复制结果，或针对当前正文重新运行。

## 删除档案

删除档案默认保留本机 Key。只有档案专属 `LTT_AI_*` 变量没有被其他档案引用时，才允许同时删除。默认档案或被作品集 Agent 使用的档案在删除前必须选择替代项。

反馈问题时，可以提供档案的 provider、model、transport、测试时间和清洗后的诊断。不要上传 API Key、数据库、私人文章、原始中转网页或本机凭据路径。
