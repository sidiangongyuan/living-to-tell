# Writer 0.2.0-alpha.45

Fixes the context pane becoming invisible or squeezed after window resize and
sidebar/context toggle operations introduced by the alpha.43 shell refresh.

## Fix

The right context pane could become invisible or squeezed down to its minimum
width (220px) after:

1. Maximizing the window, then toggling the context pane off and back on —
   the pane would stay hidden or appear much narrower than its preferred width.
2. Opening the sidebar (the fragments list) while maximized — the context pane
   would shrink to make room for the sidebar instead of the layout adapting.

**Root cause:** Multiple `QSplitter.setSizes()` call sites passed column widths
whose sum exceeded the splitter's actual width. When Qt received a sizing
request it couldn't honor (e.g., rail=80, main=2114, context=280 totaling 2474
pixels in a 2114-pixel-wide splitter), it satisfied the rail and main area's
minimum-width constraints by stealing pixels from the context pane, squeezing
it to its 220px floor or hiding it entirely.

**Fix:** The splitter normalizer now uses the splitter's real width as the
budget and never trusts the sum of input sizes. Sidebar toggle defers the
outer-splitter re-normalize to the next event loop (`QTimer.singleShot(0, ...)`)
so Qt's layout constraint propagation finishes first, ensuring the context pane
returns to its preferred width after the sidebar opens.

## Verification

- Automated offscreen repro confirms both scenarios now preserve the context
  pane at its preferred width (~280px, or the last user-dragged width).
- All 86 UI regression tests pass.

# Writer 0.2.0-alpha.45 中文说明

修复 alpha.43 界面重做后的上下文栏在窗口缩放/侧栏开合时被挤压或隐藏的问题。

## 修复

右侧上下文栏在以下场景会被挤没或收窄到最小宽度(220px):

1. 最大化窗口后,点击按钮关闭上下文栏再打开 → 栏停留在隐藏状态或显示宽度远小于偏好值。
2. 最大化时打开侧栏(碎片列表) → 上下文栏被挤窄为侧栏腾空间,而非整体布局自适应。

**根因:**多处 `QSplitter.setSizes()` 传入的三列宽度之和超过 splitter 实际宽度。Qt 收到无法兑现的尺寸请求(如:导航栏80 + 主区2114 + 上下文280 = 2474,但 splitter 实际宽2114)时,只能满足导航栏和主区的硬最小宽约束,从上下文栏偷像素,把它挤到220px底线甚至完全隐藏。

**修复:**归一化函数改用 splitter 真实宽度做预算,绝不信任传入尺寸之和;侧栏开合后延迟重算外层 splitter(`QTimer.singleShot(0, ...)`),等 Qt 的布局约束传播完成,再把上下文栏拉回偏好宽度。

## 验证

- 自动化离屏复现确认两个场景均把上下文栏保持在偏好宽度(~280px,或用户上次拖动的宽度)。
- 全部86项UI回归测试通过。
