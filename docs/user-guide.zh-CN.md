# Writer 使用教程

Writer 是一个本地优先的桌面写作工具，用来收集片段、发展长文、管理文脉资料，并在你主动选择时调用 AI 辅助。

## 1. 安装与启动

1. 从 GitHub Releases 下载最新 Windows 便携 zip。
2. 解压压缩包。
3. 打开 `Writer` 文件夹。
4. 运行 `Writer.exe`。

默认情况下，Writer 会把数据库放在系统用户数据目录中。如果你把 Writer 用于日常写作，请定期备份数据。

## 2. 第一次使用建议

1. 打开 **设置**。
2. 选择界面语言。
3. 检查编辑器舒适度设置：字号、行距、段距、正文宽度、视觉首行缩进、打字机模式。
4. 如果需要 AI，配置一个提供方：
   - OpenAI 兼容接口：使用 `env:OPENAI_API_KEY`。
   - Gemini：使用 `env:GEMINI_API_KEY` 或本机 Gemini 配置。
   - Gemini CLI / OAuth：适合已经在本机使用 Gemini CLI 的用户。
5. 第一次调用 AI 时，建议用非敏感示例文本测试。

## 3. 每日写作流程

1. 从左侧导航进入 **日期**。
2. 在日历中选择日期。
3. 新建当天片段。
4. 直接写作；Writer 会自动保存。
5. 需要时添加标签，方便以后查找。
6. 日期页顶部的每日文脉卡片会从你的文脉库中抽取一条内容，可作为当天写作提示。

## 4. 快速速记流程

快速速记适合记录突然出现的想法，不打断主窗口中的写作状态。

- 默认快速速记快捷键：Ctrl+Alt+反引号键。
- 默认主窗口显隐快捷键：`Ctrl+Alt+M`。
- 支持时，关闭主窗口会默认进入托盘。

建议先快速写下内容，稍后再整理标签、归档，或纳入长文。

## 5. 片段

片段是 Writer 的核心单位。

适合存放：

- 一句话
- 一段文字
- 一个场景种子
- 人物笔记
- 观察或记忆
- 风格练习

推荐流程：

1. 先快速记录片段。
2. 有必要时添加标签。
3. 写长文时通过搜索或筛选找回片段。
4. 如果某段文字值得反复参考，可以保存到文脉库。

## 6. 作品与作品集

使用 **作品** 写较长的文章、故事、章节或草稿；使用 **作品集** 管理一组作品。

建议流程：

1. 为一篇文章、故事或章节创建作品。
2. 需要结构时添加节块。
3. 把合适的片段纳入作品。
4. 大改之前保存手动版本。
5. 导出为 TXT、Markdown 或 DOCX。
6. 需要整理成系列或文集时，把多个作品加入作品集。

## 7. 文脉库

文脉库用于保存以后可能会复用、参考或学习的材料。

常见内容包括：

- 风格样本
- 意象
- 技法
- 人物
- 场景设定
- 原文摘录

日期页的每日文脉卡片也来自这个库。Writer 不会创建单独的精句库。

## 8. AI 工作区

AI 工作区是可选助手，不会自动接管写作。

推荐安全流程：

1. 选择作用范围：片段、选中文本、作品、作品集、文脉资料或粘贴文本。
2. 选择任务：润色、扩写、续写、总结、提纲、风格迁移等。
3. 发送前确认上下文，避免发送敏感内容。
4. 先预览 AI 输出，再决定是否应用。
5. 对重要文本执行写回前，保留版本快照。

AI 提供方会收到你主动发送的文本。敏感内容请谨慎发送。

## 9. 导出

Writer 支持导出：

- TXT
- Markdown
- DOCX

导出前建议检查作品标题、节块顺序和作品集顺序。

## 10. 备份与隐私检查

发布截图、问题反馈、日志或示例文件前，请确认：

- 使用演示文本，不使用私人写作内容。
- 不包含真实 API key 或 OAuth token。
- 不包含账号邮箱或云项目 ID。
- 不包含本地数据库文件。
- 不包含真实本机路径；如需说明路径，请使用通用示例。
