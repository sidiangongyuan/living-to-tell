# 活着为了讲述 使用教程

活着为了讲述（Living to Tell）是一个本地优先的桌面写作工具，用来写文章、编作品集、管理文脉资料，并在你主动选择时调用 AI 辅助。

## 1. 安装与启动

1. 从 GitHub Releases 下载最新 Windows 安装包。
2. 运行 `LivingToTell_0.1.9_x64-setup.exe`。
3. 从开始菜单或桌面快捷方式打开 **活着为了讲述**。

默认情况下，应用会把 SQLite 数据库保存在 `%APPDATA%\LivingToTell\LivingToTell\living-to-tell.sqlite3`。

Windows 安装器通常把程序文件放在 `%LOCALAPPDATA%\活着为了讲述`。程序文件和写作数据是分开的。

卸载应用不会删除文章数据库、备份或检查点。可在 **设置 → 数据与存储** 打开当前数据目录、打开备份目录，或把数据目录迁移到另一个本地文件夹。迁移只复制数据，旧目录会完整保留。

如果存在旧 Writer 数据库 `%APPDATA%\Writer\Writer\writer.sqlite3`，首次启动会复制到新目录；旧数据库保留不删除。

## 2. 第一次使用建议

1. 打开 **设置**。
2. 选择界面语言。
3. 检查关闭按钮行为：每次询问、最小化到托盘或直接退出。
4. 如果需要 AI，配置一个提供方：
   - OpenAI 兼容接口：使用 `env:OPENAI_API_KEY`。
   - Gemini：使用 `env:GEMINI_API_KEY` 或本机 Gemini 配置。
   - Gemini CLI / OAuth：适合已经在本机使用 Gemini CLI 的用户。
   - OpenCode：适合已经在本机运行过 `opencode auth login` 的用户。设置页可实时获取 OpenCode 模型，例如 `opencode/deepseek-v4-flash-free`、`opencode/mimo-v2.5-free`、`opencode/nemotron-3-ultra-free`、`opencode/north-mini-code-free`、`opencode/big-pickle`。
5. 先点击 **检查本地配置**，确认凭据来源存在；再点击 **发送真实测试请求**，用一条短示例请求验证 provider、model、base URL、key 和 transport。
6. 第一次正式调用 AI 时，仍建议先使用非敏感示例文本。

## 3. 每日写作流程

1. 从左侧导航进入 **日期**。
2. 在日历中选择日期。
3. 空日期可点击 **开始写作**，已有日期可打开当天文章。
4. 直接写作；应用会自动保存。
5. 需要时添加标签，方便以后查找。
6. 日期页顶部的每日文脉卡片会从你的文脉库中抽取一条内容，可作为当天写作提示。

## 4. 文章

文章是核心写作单位。

适合存放：

- 一句话发展出的短文
- 一段完整场景
- 一篇随笔或短篇
- 人物笔记
- 观察或记忆
- 风格练习

推荐流程：

1. 先快速记录文章草稿。
2. 有必要时添加标签。
3. 用文章便签记录临时提醒和下一步写法。
4. 通过搜索或筛选找回文章。
5. 如果某段文字值得反复参考，可以保存到文脉库。

## 5. 作品集

使用 **作品集** 管理一组文章。作品集不会复制或删除文章正文，只保存收录关系和顺序。

建议流程：

1. 先写好一篇或多篇文章。
2. 新建作品集。
3. 把文章加入作品集。
4. 在作品集中调整顺序，并用右侧预览检查阅读节奏。
5. 导出为 TXT、Markdown 或 DOCX。

## 6. 文脉库

文脉库用于保存以后可能会复用、参考或学习的材料。

常见内容包括：

- 风格样本
- 意象
- 技法
- 人物
- 场景设定
- 原文摘录

日期页的每日文脉卡片也来自这个库。应用不会创建单独的精句库。

## 7. AI 工作区

AI 工作区是可选助手，不会自动接管写作。

推荐安全流程：

1. 选择作用范围：文章、选中文本、作品集、文脉资料或粘贴文本。
2. 选择任务：润色、改写、扩写、续写、总结、提纲、标题等。
3. 发送前确认上下文，避免发送敏感内容。
4. 先预览 AI 输出，再决定是否应用。
5. 明确选择替换、插入或复制；AI 结果不会自动写回正文。

AI 提供方会收到你主动发送的文本。敏感内容请谨慎发送。

### AI 卡片与场景模块

AI 卡片用于保存可复用的写作上下文，而不是保存大段原文。

- **风格卡**：记录语言质感、句法节奏、叙述距离、意象偏好等。
- **人物卡**：记录角色核心、欲望、恐惧、行动模式和关系张力。
- **场景卡**：记录场景原型、触发条件、核心冲突、关键动作、情绪曲线、叙事功能、场景 DNA 和可替换元素。

新建卡片时可以插入固定模板。也可以把一段材料交给 **卡片生成 AI**，先生成草稿，确认后再保存或覆盖当前卡片。

在 AI 工具页中，场景模块需要手动搜索并勾选；未勾选的场景卡不会自动发送给 AI。

## 8. 导出

活着为了讲述支持导出：

- TXT
- Markdown
- DOCX

导出前建议检查文章标题、题记、作品集说明和作品集中文章顺序。

发布截图、问题反馈、日志或示例文件前，请确认：

- 使用演示文本，不使用私人写作内容。
- 不包含真实 API key 或 OAuth token。
- 不包含账号邮箱或云项目 ID。
- 不包含本地数据库文件。
- 不包含真实本机路径；如需说明路径，请使用通用示例。
