# 活着为了讲述官方手册

活着为了讲述（Living to Tell）是一个本地优先的桌面写作工作室，用来写文章、编排作品集、管理文脉和意象，并在你主动选择时调用 AI。这个手册按任务组织：先告诉你适合做什么，再给出具体步骤、注意事项和常见问题。

快速演示见 [GIF 教程](tutorials.zh-CN.md)。

## 1. 安装、数据与安全边界

适合做什么：确认程序文件和写作数据分开，知道卸载、升级、备份分别会影响什么。

步骤：

1. 从 [GitHub Releases](https://github.com/sidiangongyuan/living-to-tell/releases/tag/living-to-tell-v0.1.49) 下载最新 Windows 预览版。
2. 推荐运行 `LivingToTell_0.1.49_x64-setup.exe`；也可以使用 `LivingToTell_0.1.49_x64_zh-CN.msi`。
3. 从开始菜单或桌面快捷方式打开 **活着为了讲述**。
4. 进入 **设置 → 数据与存储**，查看当前 SQLite 数据库、备份目录和检查点目录。
5. 大篇幅修改前，进入 **导出与备份** 创建一次备份或命名检查点。

注意事项：

- 写作数据库默认保存在 `%APPDATA%\LivingToTell\LivingToTell\living-to-tell.sqlite3`。
- 程序通常安装在 `%LOCALAPPDATA%\活着为了讲述`，这和写作数据库不是同一个位置。
- 卸载应用不会删除文章数据库、备份或检查点。
- 数据目录迁移采用复制方式，旧目录会完整保留。
- 如果存在旧 Writer 数据库 `%APPDATA%\Writer\Writer\writer.sqlite3`，首次启动会复制到新目录；旧数据库不删除。

常见问题：

- **Windows 提示未签名风险**：当前预览版暂未签名，只运行本仓库 Release 页下载的安装包。
- **升级后数据会不会没了**：正常升级不会删除写作数据库。升级前仍建议创建检查点。
- **要不要手动复制数据库**：可以，但更推荐用应用内备份/检查点，路径和恢复入口更清楚。

## 2. 首次使用与示例项目

适合做什么：第一次打开时快速理解完整工作流，而不污染自己的真实作品库。

步骤：

1. 打开 **日期** 页。
2. 查看欢迎清单：第一篇文章、第一条文脉、AI 设置、备份位置、文章对话。
3. 如果想快速体验完整流程，点击 **创建示例项目**。
4. 点击 **打开作品集**，查看示例文章、作品集书稿结构、文脉标本、文章便签和场景 AI Card。
5. 体验结束后，回到日期页点击 **删除示例**。

注意事项：

- 示例项目不会自动创建，必须手动点击。
- 删除示例项目只删除应用记录在 marker 中的精确 ID，不按标题或标签模糊删除。
- 你自己创建的同名文章、同名标签、同名作品集不会因为删除示例而被清理。

常见问题：

- **欢迎清单不见了**：进入 **设置**，使用“重新显示欢迎清单”。
- **示例项目是否能长期保留**：可以，它只是普通本地内容；不需要时再删除。

## 3. 文章写作

适合做什么：写随笔、短篇、场景、人物笔记、观察记录和长文草稿。

步骤：

1. 从 **日期** 页点击空日期的 **开始写作**，或从左侧导航进入 **文章**。
2. 在左侧列表选择文章，右侧是正文编辑区。
3. 直接输入正文；应用会自动保存。
4. 给文章添加标签，方便以后搜索和筛选。
5. 在右侧上下文栏写文章便签，用来记录临时提醒、下一步写法或删掉的片段。
6. 打开 **历史版本**，在大改前点击 **保存当前版本**。
7. 需要沉浸写作时进入专注模式，只保留正文区域。

注意事项：

- 文章便签不进入正文、导出和全文搜索。
- 历史版本只保存关键节点：手动保存、AI 写回前、恢复前。
- AI 结果不会自动写回正文，必须你明确选择替换、插入或复制。

常见问题：

- **正文开头缩进会不会丢**：当前保存逻辑会保留正文开头的全角空格。
- **切换文章偶尔显示 Not Found**：旧请求会被忽略；如果当前文章真的不存在，应用会刷新列表并显示中文提示。
- **长文末尾写作不舒服**：编辑器保留文末空白，方便在底部继续写。

## 4. 作品集与书稿结构

适合做什么：把多篇文章组织成一本书稿项目。作品集现在只有一条主线：**书稿结构**。它同时负责层级、顺序、规划状态、关联文章和导出。

步骤：

1. 进入 **作品集**。
2. 第一次进入只会显示轻量邀请：点 **开始教程** 才进入逐步高亮，点 **稍后** 下次仍可提示，点 **不再提示** 才长期关闭。作品集和其他复杂工作台的教程都可在 **设置 > 界面与教程 > 教程中心** 重新启动。
3. 点击 **新建作品集**，填写标题、说明，并选择作品类型：通用、小说、散文集 / 文集、非虚构。创建后需要改标题或简介时，点顶部 **编辑简介**。
4. 在 **书稿结构** 中新增顶层节点，例如分部、章节、辑、篇章或小节。具体叫法会随作品类型变化。
5. 如果一个章节下面需要多篇文章，先选中章节，再点 **新子项**，或把 **未编排文章** 放到当前节点下。
6. 一个结构节点直接关联一篇文章；需要多篇文章时，用多个子项分别关联。
7. 选中节点时先阅读摘要、状态、关联正文和规划信息；需要修改时点 **编辑详情**，再设置状态、摘要、视角、时间线、地点、标签和目标字数。
8. 切到 **看板**，按构思、草稿、修订、完成、暂停查看同一棵书稿结构的推进状态。
9. 需要时从结构节点一键创建文章，或把现有文章关联到节点。
10. 导出作品集为 Markdown、TXT 或 DOCX；导出优先读取书稿结构。

注意事项：

- 作品集只保存关系、结构和导出顺序，不复制正文。
- 从作品集中移除文章不会删除原文章。
- 在文章页右侧的 **所属作品集** 中点击 **打开**，可以直接回到对应作品集并定位这篇文章。
- 未编排文章已经属于作品集，但还没有进入书稿结构；当作品已有结构关联文章时，它们不会自动进入正文导出。
- “标题”是这个结构节点的名字；“类型”表示层级或用途；“上级节点”决定它在树中的位置；“关联文章”只建立连接，不复制或移动正文。
- 作品类型只改变术语：小说偏“分部 / 章节 / 场景”，散文集偏“辑 / 篇组 / 篇章”，非虚构偏“部分 / 章节 / 小节”。

常见问题：

- **一个章节下怎么放多篇文章**：章节本身可以关联一篇文章，也可以包含多个子项。选中章节后新增子项，或把未编排文章放到该章节下。
- **可以只规划不写正文吗**：可以。结构节点可以先不关联文章。
- **“分部”和“部分”哪个对**：小说里用“分部”更像文学结构；非虚构里用“部分”更像说明性结构。作品类型会自动切换叫法。

### 用作品集 Agent 一起构思和写候选稿

Agent 绑定当前作品集。它可以从零和你构思小说、讨论不同方向、把决定整理成结构、写场景候选稿、检查连续性；但不会把普通对话当成 canon，也不会擅自把草稿写进正文。

顶部的 **会话** 按钮控制左侧会话与 Prompt 索引；**工作栏** 控制右侧的上下文、草稿、待确认提案和项目圣经。它不是“导入资料”按钮。两栏都可以折叠；普通桌面宽度下工作栏以抽屉覆盖，不会继续压缩中间对话区。

1. 打开作品集的 **Agent** 标签，为一条明确工作线新建命名会话，例如“第一幕构思”或“主角弧光”。
2. 选择 **讨论 / 规划 / 草稿 / 审校**。顶部 AI 配置只覆盖下一轮；作品集默认模型在 **工作栏 > 上下文** 中设置。
3. 直接用自然语言交流。只有需要精确材料时才输入 `@`，加入文章、结构节点、AI 卡片、意象或文脉；“上下文”栏会明确显示下一轮能读什么。
4. 草稿模式只填真正有用的场景约束：目标场景、视角、时态、大致长度、必须发生和需要避免的内容。
5. 生成的正文先进入本机 **草稿库**。可以编辑、要求一个变体，也可以离开页面后再回来处理。
6. 应用草稿时可选择新建文章、追加到已有文章，或替换一段明确且唯一的选区。写入已有文章前会检查正文漂移并自动创建版本快照。
7. 只有作者确认的书稿事实才进入 **项目圣经**。普通对话、会话工作摘要、被拒绝提案和未应用草稿都不会进入长期记忆。
8. 作者风格闭环从你的章节原稿开始，在共同讨论、修改并由你标记章节完成时结束。每积累 3 个新闭环，软件只在本地提示可生成作者画像更新提案，不会自动调用模型。

模型提出项目圣经更新时，软件会把常见的中英文栏目名称归一为固定栏目。无法可靠识别的栏目不会被静默写入；应用失败时会显示中文说明，拒绝提案不会改变项目圣经。

离开 Agent 标签不会取消运行中的任务；重新打开时只恢复任务状态，不会重新发送供应商请求。点击 **中断** 只停止本机等待，已经发出的远端请求仍可能完成或产生费用。

## 5. 文脉库与意象星图

适合做什么：保存以后会反复参考的摘录、风格样本、意象、技法、人物材料和来源信息。

步骤：

1. 进入 **文脉库**。
2. 新建文脉标本，填写正文、书名/篇名、作者、用途和个人笔记。已有标本默认是阅读态；点 **编辑** 后才显示可修改字段。
3. 搜索会覆盖标题、作者、标签、个人笔记和正文，并高亮命中词；不在编辑器中时可用上下键移动、Enter 打开，Escape 退出编辑态。
4. 在日期页的每日文脉卡片中，可以跳回对应标本。
5. 在文章或文脉正文中选中文字后右键，选择 **加入意象星图**。
6. 在弹窗中搜索已有意象或创建新意象，保存摘录。
7. 进入 **意象星图**。滚轮缩放、拖动画布，使用适配或居中，调整显示密度；点击节点会聚焦一跳邻域。
8. 在 **正式关系** 中手动连接两个已有意象，类型可选呼应、对照、转化、包含、一般关联；每对意象只保存一条可编辑关系。
9. 需要 AI 帮忙寻找关系时点 **发现关联**。AI 只读取当前意象档案和最多 200 个意象的轻量索引；已有关系候选与新概念候选默认都不勾选。只有选中并应用后才写入，新概念只创建带名称的待丰富空节点。
10. 在意象详情点击 **AI 丰富** 生成概念档案；预览后再保存。丰富候选和关系候选都不会自动写库。
11. 在意象详情中移除摘录时，只会解除当前意象链接；同一摘录在其他意象中仍保留。

注意事项：

- 文脉库保存来源和用途；意象星图保存“这句话属于哪些反复出现的意象”。
- 同一来源同一选区会复用同一摘录，不重复创建。
- 如果文章修改导致位置漂移，应用会尝试用文本和上下文找回原摘录并合并重复项。
- 正式关系由作者确认，与真实摘录共现共同显示；AI 候选在选择并应用前不入库，确认后的关系也不会标成“AI 边”。
- 同一对意象同时存在共现和正式关系时只画一条合并边；删除其中一种来源，只要另一种仍存在，边就保留。

常见问题：

- **同一句话能加入多个意象吗**：可以，一条摘录可以同时连接多个意象。
- **删除一个意象里的摘录会不会删掉其他意象里的同一句话**：不会，默认只是从当前意象移除。

## 6. AI 修改、文章对话与 AI Cards

适合做什么：对一篇真实文章或明确选区做润色、改写、扩写、续写；在文章侧抽屉讨论写法；用 AI Cards 保存可复用的风格、人物和场景指导。作品集层面的讨论与草稿继续交给作品集 Agent。

步骤：

1. 进入 **设置 → AI**，用三步向导新建档案，或扫描本机 Codex、Gemini、OpenCode 配置。
2. 选一个档案设为默认。单模型功能使用这个档案，除非功能里提供了明确的临时选择器。
3. 先运行 **全部本地检查**，只检查本机凭据与登录状态，不联系模型。
4. 只勾选要验证的档案，再发送最小真实测试。真实测试可能使用 token 并产生费用；档案会保存最近健康状态与测试时间。
5. 在文章页点击 **AI 修改**；也可以先选中文字再进入。直接打开 AI 修改页时需要先选文章，不再提供粘贴任意文本模式。
6. 选择润色、改写、扩写或续写。
7. 在一级 **参考上下文** 中按需组合 **文脉标本 / AI Cards / 文章便签**。三类资料都有大尺寸搜索选择器、清晰卡片、全文预览、草稿多选和固定确认栏。
8. 文脉标本可按用途筛选，AI Cards 可按风格 / 人物 / 场景筛选，文章便签可按进行中 / 已完成筛选且置顶优先。切换文章会清空便签选择，避免把上一篇便签发给模型；文脉和 Cards 在本次 AI 修改页面会话中保留。
9. **更多要求** 只保留预设、补充要求和高级任务参数，不再隐藏三类参考资料。
10. 按需选择一个或多个档案。只选默认档案时第一次勾选其他档案会替换默认项；只有你再次手动勾选，默认档案才会加入多模型任务。
11. 运行后，第一个成功结果立即可读；某个模型失败只更新自己的状态，不阻塞其他结果。离开页面不会重发任务。
12. 在 **生成结果 / 与原文差异** 间切换。结果会记录冻结上下文的名称和字数，不回传附件正文；写回必须先预览，系统会检查文章是否漂移，并先创建 `AI_BEFORE_APPLY` 版本。
13. 在文章页打开 **AI 对话** 进行讨论。关闭抽屉后保留输入、会话和等待中的回复；复制或保存回复仍需明确点击。
14. 进入 **AI Cards** 创建风格、人物或场景卡；AI 生成的卡片先预览再保存，需要参与文章修改时直接在 AI 修改选择器中勾选。

注意事项：

- AI 提供方只会收到你明确运行任务时所示的文章或选区，以及你主动勾选的上下文。
- “检查本地配置”不是远端可用证明；只有“发送真实测试请求”成功才代表模型可用。
- 多模型请求只包含实际勾选的档案 ID，默认档案不会被暗中补入。
- 文脉标本、AI Cards 和文章便签都只有在各自选择弹窗确认后才会附加；同一轮所有模型收到完全相同的冻结选择，运行元数据不会把附件正文再次返回前端。
- 文脉标本只用于借鉴写法、风格、结构、意象或修辞，不是事实来源。提示词明确禁止照搬或近似改写原句，也禁止移植标本中的人物、事实、情节和专有名词。
- 重连只查询任务状态，不重新发送 provider 请求；本地中断也不能保证已发出的远端请求停止生成或停止计费。
- 文章内容发生漂移时会阻止位置写回；应复制结果，或针对当前正文重新运行，不能强行覆盖错误位置。
- 场景模块和其他上下文需要手动勾选；未勾选的材料不会发送给 AI。
- AI 卡片适合保存结构，不适合保存大段原文；最好保持可读，之后才能作为提示词上下文复用。

常见问题：

- **OpenCode 是否需要在应用里保存 key**：不需要，应用调用本机 `opencode auth login` 后的 CLI 登录状态。
- **能不能在应用内登录 ChatGPT**：不能。OpenAI 官方接入使用 API Key；Codex 接入复用本机已有 Codex 登录。
- **为什么本地超时后中转站仍扣了 token**：请求发出后，provider 可能继续处理。应用重连不会自动重发，因此不会因为查看状态而重复扣费。
- **AI 结果会不会自动覆盖文章**：不会，写回必须你明确确认。
- **特别长的标本为什么只发送一部分**：单个附件有 40,000 字发送上限。AI 修改会先保留用途、标签和作者备注，再发送上限内的正文；文脉库中的完整标本不会被修改。

按状态码和接入方式排查时，见 [AI 配置排查](ai-troubleshooting.zh-CN.md)。

## 7. 导出与备份中心

适合做什么：在大改前留恢复点，把文章或作品集导出成外部文件，并确认数据路径。

步骤：

1. 进入 **导出与备份**。
2. 查看安全摘要：最近恢复点距离现在多久、备份数量、检查点数量、总大小。
3. 大改前点击 **创建检查点**，写一个能看懂的名字和说明。
4. 也可以点击 **创建备份**，生成自动备份。
5. 在恢复点列表中选择一个备份或检查点。
6. 如果需要恢复，点击 **恢复所选**；应用会在恢复前自动备份当前数据库。
7. 在 **数据位置** 中复制或打开数据库、备份、检查点目录。
8. 在导出快捷操作中导出最近文章或最近作品集。

注意事项：

- 恢复入口只使用应用管理的备份或检查点。
- 恢复前会自动备份当前数据库，降低误恢复风险。
- 路径显示用于确认和复制，不会修改数据。

常见问题：

- **多久备份一次**：高频写作建议每天或每 3 天；普通使用每 7 天也可以。
- **导出和备份有什么区别**：导出是给人读的 TXT/Markdown/DOCX；备份是给应用恢复数据库用的 SQLite 副本。
