## Reference specimen regressions

- Purpose filtering and FTS search cover title, author, specimen text, tags, and personal notes.
- Polish, Rewrite, Expand, and Continue all receive specimen anti-copy rules; ordinary references, AI Cards, and notes do not trigger those rules.
- The picker stages changes until confirmation, retains choices across search and purpose filters, supports full-text preview, and resets only after leaving or refreshing AI Edit.
- Every selected model receives the same frozen specimen context. Later UI changes affect only a later run.
- Task snapshots expose only kind, reference ID, display name, and the actual bounded attachment character count; response payloads never repeat specimen text.
- Background result recovery identifies the references used even after the active picker selection has reset on navigation.
- The picker passes serious/critical Axe checks and remains within `1024x768`, `1400x900`, and `1920x1080` viewports in Chinese and English, with no horizontal overflow.

## Real-provider tests

Only a disposable synthetic article and reference specimen were used. No user article, saved reference, database content, screenshot, or Collection Agent memory was sent.

An initial dual-model run exposed that the OpenAI-compatible SDK could wait for roughly ten minutes and perform internal retries. No application-level reconnect resent that request. The provider path was then bounded to 120 seconds by default and SDK retries were disabled; a new paid request can now happen only after the author explicitly starts another run.

One explicit post-fix DeepSeek + GLM run produced independent model outcomes:

| Model | Result | Transport | Elapsed | Tokens |
| --- | --- | --- | ---: | ---: |
| `deepseek-v4-pro` | Success | `openai_chat_completions` | 14.134s | 424 input / 426 output |
| `glm-5.2` | Safe timeout | `openai_chat_completions` | 121.208s | Not reported |

The GLM timeout was not retried again. Its public error correctly warned that the remote provider might still be generating or billing. The DeepSeek result remained usable, so one model's failure did not block another model's success. Both models received the same frozen `style_specimen` snapshot, status responses did not expose specimen text, and all disposable rows were removed afterward. Business-table counts returned exactly to their baseline.
