# Living to Tell Preview 0.1.39

This update focuses on making the Collection Agent easier to use during long manuscript work.

## Highlights

- The Agent reference search is now a compact picker instead of a long always-open panel. Use `+ Add Reference` or type `@` in the prompt box, pick a result, and it becomes a context chip.
- Quick tasks now ask for confirmation before sending any provider request. The confirmation shows the task, selected model, context scope, and prompt preview so accidental clicks do not spend tokens.
- Agent sessions now have a prompt index. It lists your prompts and quick tasks, supports search, and jumps to the matching result card.
- Long Agent answers collapse by default, so the workspace stays navigable.
- Articles can now jump directly to an owning collection from the article context pane.
- `/clear` and the Clear conversation button clean the Agent work log without deleting the project bible, article data, outline data, or pending proposals.

## Safety Notes

- Clearing a Collection Agent session does not write to or erase long-term project memory.
- Rejected proposals still do not enter memory.
- Project memory changes only when you save the project bible manually or apply an `update_memory` proposal.
